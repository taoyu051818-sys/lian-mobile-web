# 已确认的 NodeBB 对接逻辑

| 发布类型    | 前端/业务入口                 | 后端 handler         | helper                                                  | NodeBB endpoint              | 发布用户                            |
| ------- | ----------------------- | ------------------ | ------------------------------------------------------- | ---------------------------- | ------------------------------- |
| 普通发布    | `handleCreatePost()`    | 普通发帖逻辑             | `createNodebbTopicFromPayload(auth, payload)`           | `POST /api/v3/topics`        | 当前登录用户对应的 `nodebbUid`           |
| AI 发布   | `handleAiPostPublish()` | AI 发帖逻辑            | `createNodebbTopicFromPayload(auth, normalizedPayload)` | `POST /api/v3/topics`        | 可使用配置中的用户 uid，也可测试时用管理员 `uid=2` |
| 管理员测试发布 | Codex / 测试入口            | AI publish 或后台发布逻辑 | 同上                                                      | `POST /api/v3/topics?_uid=2` | 管理员账号                           |

核心点是：

```text
普通发帖和 AI 发帖最终都走同一个 NodeBB 创建 topic 接口。
区别只在于 payload 来源不同，以及使用哪个 nodebbUid 发布。
```

# 可以给团队的简洁版本

目前项目已经实现了基础社区能力，包括登录、发帖、浏览、点赞、收藏、评论、匿名投稿和 AI 发帖发布。
NodeBB 目前主要作为论坛底座使用，普通发帖和 AI 发帖都通过 `POST /api/v3/topics` 创建 topic，并通过 `_uid=<nodebbUid>` 指定发布用户。
AI publish 可以继续使用配置里的用户 uid 发布，测试阶段可以使用已经赋权的管理员账号，例如 `uid=2`。

接下来最邻近的开发重点应该是：

| 优先级 | 功能                        |
| --- | ------------------------- |
| P0  | 注册、学生认证、分类体系、图片上传、内容审核    |
| P1  | 拼车模板、闲置模板、食堂菜单卡片、AI 发布确认页 |
| P2  | 首页信息流、热门推荐、搜索、通知系统        |
| P3  | 地图化信息、区域频道、AI 每日摘要、校园知识库  |

整体方向是：

```text
先把 NodeBB 作为稳定内容底座跑通
再用 AI 做信息整理和内容运营
最后逐步升级成地图化、结构化、可搜索的校园信息系统
```
