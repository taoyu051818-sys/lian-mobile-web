# LIAN Campus UI / UX Guidelines V0.1

LIAN 不是论坛加地图，而是以地点为归宿的校园信息系统。

卡片是信息的容器，马甲是用户的身份，地点是内容的归宿。

视觉让它统一，动效让它连贯，状态让它可信，身份让它安全，地点让它沉淀。

## 1. 产品设计原则

1. 信息优先于装饰。
2. 信息流优先于功能宫格。
3. 卡片承载信息。
4. 马甲承载身份。
5. 地点组织沉淀。
6. 算法调节内容生态。
7. 玻璃只做浮层材质。
8. AI 只做整理和辅助，不做事实权威。
9. 主操作永远比次操作明显。
10. 内容要可搜索、可归档、可复用。

LIAN 的 UI 判断优先级是：内容 > 身份 > 地点 > 来源与状态 > 操作 > 装饰。

## 2. 视觉基础

LIAN 的视觉方向是 Campus Utility Glass：校园生活工具感 + 滨海轻玻璃感。

Utility 是核心，Glass 是材质。工具性优先，玻璃感辅助。

颜色可以辅助识别，但不能成为唯一语义。

## 3. 玻璃 UI 与层级

玻璃不是风格主角，玻璃是层级工具。

| 类型 | 用途 | 规则 |
| --- | --- | --- |
| Glass Bar | 顶部栏、底部导航 | 轻 blur，低阴影 |
| Glass Panel | 地图详情卡、发布表单、底部抽屉 | 中 blur，中阴影 |
| Glass Orb | 返回、关闭、定位、发布悬浮按钮 | 圆形，识别强 |
| Tonal Pill | 保存草稿、重新生成、筛选 | 浅色、低强调 |
| Filled Pill | 发布、确认、下一步、提交 | 实色、高强调 |

决定流程结果的按钮，不能只是玻璃按钮。

## 4. Card System 卡片系统

统一卡片底色，轻量类型色，结构优先，颜色辅助。

LIAN 信息流不使用大面积类型色。卡片整体保持统一底色。内容类型通过 Type Chip、Icon、轻色条、来源状态和内容模块区分。

```txt
[Type Chip] [Location Chip] [Trust Badge]

标题

摘要 / 图片 / 特定内容模块

IdentityBadge · ContributionBadge
时间 · 评论 · 收藏 · 沉淀状态
```

V0.1 先收敛成 4 个母卡片：ContentCard、PlaceCard、ActionCard、SummaryCard。

## 5. 身份、贡献标签与 Tag Chip

标签是内容连接系统，不是装饰。

规则：展示时必须带 #；用户输入时 # 可选；系统保存时统一补 #；最多 5 个；单个标签正文最多 15 个字符；允许中文、英文、数字、下划线、连字符。

身份展示规则：卡片上最多展示 1 个身份标签 + 1 个贡献标签；发布页必须显示当前发布身份；评论框必须显示当前评论身份。

## 6. 信息流、地点与地图沉淀

信息流负责分发，地图负责定位，地点页负责沉淀。

内容生命周期：

```txt
内容 → 信息流分发 → 地点归档 → 被搜索 / 地图 / AI 复用
```

地图默认展示稳定地点、当前筛选、附近高相关、少量热帖和用户正在操作的内容。不要默认展示所有帖子。

## 7. 动效、交互、状态与反馈

动效必须解释关系，而不是制造热闹。

规则：页面转场只用 transform / opacity；不动画 width / height / top / left；长列表不要每次大面积 reveal；玻璃 blur 不参与强动画；必须支持 prefers-reduced-motion。

事件优先级：按钮 > chip > 图片 > 身份 > 卡片主体。

所有写操作必须有即时反馈。失败时必须告诉用户下一步。

## 8. 文案、无障碍与数据可信度

LIAN 文案气质：清楚、友好、不装、不官方腔、不制造压力。

无障碍规则：按钮点击区域不小于 44px；重要文字不能低对比；图标按钮必须有 aria-label；不能只靠颜色区分状态；地图 marker 需要文字或类型图标。

每条可沉淀信息都应能回答：谁提供的、什么时候更新的、是否被确认、是否来自 AI、是否过期、能不能查看来源。

没有来源的 AI 摘要，只能作为草稿，不能作为事实卡片。

## 9. 页面评审 Checklist

1. 主信息是否足够清楚？
2. 是否使用统一卡片结构？
3. 主操作是否明显？
4. 玻璃 UI 是否影响阅读？
5. 是否显示当前身份？
6. 是否使用标签 chip？
7. 是否绑定地点？
8. 是否能沉淀到地点页或资料库？
9. AI 内容是否标注来源和不确定性？
10. 错误状态是否告诉用户下一步？
11. 状态是否不只依赖颜色？
12. 触控区域是否足够？
13. 是否支持减少动态效果？
14. 是否符合移动端优先？
