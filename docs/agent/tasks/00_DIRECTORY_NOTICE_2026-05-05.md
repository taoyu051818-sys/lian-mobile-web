## Safe usage

Use task docs to understand old scope, allowed-file intent, acceptance ideas, risks, and rollback notes.

Do not use task docs alone to decide whether a task is still active, whether a feature is missing, whether a repo owns a file today, or what validation command should be run today.

If resuming an old task, create a dated addendum or fresh task after revalidating against current code and merged PRs.
